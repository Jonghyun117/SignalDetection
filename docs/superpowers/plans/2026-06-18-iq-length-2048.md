# IQ Length 1024→2048 Migration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Change IQ sample length from 1024 to 2048 throughout all C inference code, Python training code, and tests, then retrain the model.

**Architecture:** The AMCNet model uses AdaptiveAvgPool1d so weights don't change shape — only the C hardcoded intermediate sizes change. nn_forward.c buffer math must be recalculated for 2048 input.

**Tech Stack:** Python/PyTorch (training), C11 (inference), pytest

---

## Intermediate size math for nn_forward.c (2048 input)

| Layer | Input | Conv out | Pool out |
|-------|-------|----------|----------|
| L0: Conv(3→32,k=7)+Pool2 | (3,2048) | (32,2042) | (32,1021) |
| L1: Conv(32→64,k=5)+Pool2 | (32,1021) | (64,1017) | (64,508) |
| L2: Conv(64→128,k=5)+Pool2 | (64,508) | (128,504) | (128,252) |
| L3: Conv(128→128,k=3) | (128,252) | (128,250) | — |

BUF_A = 66000 (≥ 32×2042=65344)
BUF_B = 33000 (≥ 32×1021=32672)
